Tiny Assistant

The tiny assistant is the useful target application. The lab remains separate so code changes can still be tested with controlled tasks.

Executable

bin/chat is the assistant-facing executable. It loads a checkpoint and formats each user input as a canonical single-turn instruction prompt. 

./bin/chat --checkpoint artifacts/checkpoints/model.bin --greedy

Minimum Useful Scope

  • Single-turn chat remains the default behavior until it is reliable on curated examples.
  • Training data should use the canonical instruction format or JSONL import path.
  • Evaluation should include exact checks for deterministic facts and prefix or contains checks for wording-tolerant answers.
  • Eval reports should be written to artifacts/evals/ when comparing checkpoints.
  • Runtime controls should stay small: checkpoint path, token limit, temperature, top-k, greedy mode, and exit commands.

Assistant Seed Data

The tracked starter dataset lives at sample_data/assistant/seed.jsonl. Generated canonical train and validation files should live under artifacts/datasets/assistant/. 

./bin/mgpt import-jsonl \
  --input sample_data/assistant/seed.jsonl \
  --output artifacts/datasets/assistant/all.txt

./bin/mgpt validate-data \
  --input artifacts/datasets/assistant/all.txt

./bin/mgpt split-data \
  --input artifacts/datasets/assistant/all.txt \
  --train artifacts/datasets/assistant/train.txt \
  --val artifacts/datasets/assistant/val.txt \
  --ratio 0.9 \
  --seed 42

Baseline Training

Run the assistant training script to import, validate, split, train, and write train and validation eval reports. 

./scripts/train_assistant.sh

The script uses the same defaults as the explicit command below. Override settings with environment variables such as STEPS=500, D_MODEL=128, or ASSISTANT_CHECKPOINT=artifacts/checkpoints/assistant-big.bin. The script defaults to ASSISTANT_TOKENIZER=bpe; set ASSISTANT_TOKENIZER=byte only when creating a legacy byte-tokenized checkpoint. Use ASSISTANT_BACKEND=metal to build and select the Metal backend. The scripts default to ASSISTANT_SPLIT_RATIO=0, which means they train and evaluate on the full imported dataset instead of splitting out a validation file. Set it to something like 0.9 when you want a held-out validation split.

To continue training an existing checkpoint, use ./scripts/resume_assistant.sh with the same environment variables. 

./bin/mgpt train \
  --input artifacts/datasets/assistant/train.txt \
  --val-input artifacts/datasets/assistant/val.txt \
  --checkpoint artifacts/checkpoints/assistant-seed-bpe.bin \
  --steps 3000 \
  --context 128 \
  --d-model 64 \
  --layers 2 \
  --heads 4 \
  --ff 128 \
  --batch-size 8 \
  --lr 0.0008 \
  --tokenizer bpe \
  --eval-interval 500 \
  --save-interval 1000 \
  --progress-interval 500

Start Chat

Start the assistant chat executable with the current assistant checkpoint. In a terminal it opens the scrollback-based TUI with an editable input area, generation progress, and a footer showing tokens and model details: 

./scripts/chat_assistant.sh
  • Scrollback sits above the input bar so the conversation stays visible.
  • Progress updates appear while tokens are being generated.
  • The footer shows prompt tokens, completion tokens, backend, and checkpoint.
  • When stdin or stdout is not a terminal, chat falls back to line mode.

Override the checkpoint or generation settings with environment variables: 

ASSISTANT_CHECKPOINT=artifacts/checkpoints/assistant-seed-bpe.bin \
ASSISTANT_BACKEND=metal \
MAX_NEW_TOKENS=80 \
TEMPERATURE=0.2 \
TOP_K=1 \
./scripts/chat_assistant.sh

Set GREEDY=0 to use sampling with the configured TOP_K value.

Current Baseline Status

The first seed baseline trained for 3000 steps with context=128, d-model=64, and layers=2. It reached 12.94% train contains accuracy and 0.00% validation contains accuracy, so it is not ready for session context. The next experiments should improve single-turn memorization before broadening the assistant behavior. For tiny-data debugging, keep ASSISTANT_SPLIT_RATIO=0 to measure exact-match accuracy on the full imported dataset instead of a held-out split.

Session Memory

The interactive chat shell now keeps a running session history and trims older whole turns when the prompt would exceed the context window.

  • Use the same <BOS>, <USER>, <ASSISTANT>, and <EOS> markers for the whole session.
  • Trim the oldest whole turns first.
  • Use /reset to clear the current conversation.

The planned multi-turn training format stays the same shape and is now supported by the data pipeline: 

<BOS><USER>
first user turn
<ASSISTANT>
first assistant turn
<USER>
second user turn
<ASSISTANT>
second assistant turn
...
<EOS>

Training should keep the most recent whole turns that fit inside the context window. The last assistant turn remains the target to learn, and generation still stops on <EOS>.

Use --format session with import-jsonl, validate-data, and split-data to keep the session blocks intact.

A tracked sample lives at sample_data/assistant/session_seed.jsonl and can be trained with: 

ASSISTANT_SEED=sample_data/assistant/session_seed.jsonl \
ASSISTANT_FORMAT=session \
./scripts/train_assistant.sh

Quality Reporting

Assistant quality should start with mechanical checks because they are repeatable. Exact, prefix, and contains scoring are appropriate for the current model. Free-form quality review can be added later, but it should not replace deterministic regression checks.
