Experiment Workflow

Use a repeatable train/eval loop so changes in code, data, and model size can be compared cleanly.

0. Validate the Format Contract

Before training, run validate-data on the canonical instruction file. This catches malformed markers, empty examples, and structural drift before the model sees the data.

1. Generate and Split Data

./bin/mgpt make-arithmetic-data \
  --output /private/tmp/microgpt-arithmetic.txt \
  --max-a 12 \
  --max-b 9

./bin/mgpt validate-data \
  --input /private/tmp/microgpt-arithmetic.txt

./bin/mgpt split-data \
  --input /private/tmp/microgpt-arithmetic.txt \
  --train /private/tmp/microgpt-train-small.txt \
  --val /private/tmp/microgpt-val-small.txt \
  --ratio 0.9 \
  --seed 42

2. Train With an Explicit Validation File

./bin/mgpt train \
  --input /private/tmp/microgpt-train-small.txt \
  --val-input /private/tmp/microgpt-val-small.txt \
  --checkpoint /private/tmp/microgpt-baseline.bin \
  --steps 3000 \
  --context 64 \
  --d-model 64 \
  --layers 2 \
  --heads 4 \
  --ff 128 \
  --batch-size 8 \
  --lr 0.0008 \
  --eval-interval 500 \
  --save-interval 1000 \
  --progress-interval 500

3. Evaluate Exact-Match Accuracy

./bin/mgpt eval \
  --checkpoint /private/tmp/microgpt-baseline.bin \
  --input /private/tmp/microgpt-train-small.txt \
  --max-new-tokens 20 \
  --match exact \
  --greedy \
  --hide-failures

./bin/mgpt eval \
  --checkpoint /private/tmp/microgpt-baseline.bin \
  --input /private/tmp/microgpt-val-small.txt \
  --max-new-tokens 20 \
  --match exact \
  --greedy \
  --hide-failures

Use --match exact for arithmetic, JSON, and formatting tasks. Use prefix or contains for assistant examples where the generated answer may include extra useful text.

The round-trip tests exercise the same path end to end: canonical text validation, JSONL import, and deterministic splitting.

Assistant Baseline Loop

Use the tracked assistant seed data when testing the tiny assistant path. Keep generated files under artifacts/. 

./scripts/train_assistant.sh

The script runs the full import, validation, split, train, and eval pipeline. Use environment variables to override defaults, for example STEPS=500 ./scripts/train_assistant.sh or ASSISTANT_BACKEND=metal ./scripts/train_assistant.sh. 

./bin/mgpt import-jsonl \
  --input sample_data/assistant/seed.jsonl \
  --output artifacts/datasets/assistant/all.txt

./bin/mgpt split-data \
  --input artifacts/datasets/assistant/all.txt \
  --train artifacts/datasets/assistant/train.txt \
  --val artifacts/datasets/assistant/val.txt \
  --ratio 0.9 \
  --seed 42

./bin/mgpt train \
  --input artifacts/datasets/assistant/train.txt \
  --val-input artifacts/datasets/assistant/val.txt \
  --checkpoint artifacts/checkpoints/assistant-seed.bin \
  --steps 3000 \
  --context 128 \
  --d-model 64 \
  --layers 2 \
  --heads 4 \
  --ff 128 \
  --batch-size 8 \
  --lr 0.0008 \
  --eval-interval 500 \
  --save-interval 1000 \
  --progress-interval 500

./bin/mgpt eval \
  --checkpoint artifacts/checkpoints/assistant-seed.bin \
  --input artifacts/datasets/assistant/val.txt \
  --max-new-tokens 80 \

  --greedy \
  --output artifacts/evals/assistant-seed-val.json
