Engineering Policy

Formatting

Source formatting is defined by .clang-format. The project uses C++17, two-space indentation, and a 120-column limit.

Hardening Builds

make test
make cli-test
make strict-test
make sanitize-build

strict-test treats warnings as errors. sanitize-build compiles all targets with address and undefined-behavior sanitizers.

Input Limits

File reads through read_file_bytes and read_file_text are capped at 256 MiB. The byte tokenizer requires vocab_size >= 256.

Checkpoint Compatibility

Checkpoints use the MICROGPT1 binary format and store CPU float32 weights plus optimizer state. Backend-specific device buffers are not serialized. A checkpoint trained with Metal must load and generate on CPU; this is covered by metal_checkpoint_cpu_generation_interop_test.

Header State

Most implementation currently remains header-only. Static state in inline functions is allowed only for process-wide runtime caches such as backend transfer counters or Metal runtime objects. New public integration should use microgpt/api.hpp or microgpt/api_c.h instead of internal headers.
