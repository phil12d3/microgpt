Data Format

microgpt currently uses a compact single-turn instruction format. The tokenizer treats the control markers as special tokens and every other character as a byte token.

Single-Turn Instruction Examples

Each example should use this shape:

<BOS><USER>
What is 2+2?
<ASSISTANT>
4
<EOS>

Markers

  • <BOS> marks the beginning of an example.
  • <USER> marks the user prompt.
  • <ASSISTANT> marks where the answer begins.
  • <EOS> marks the end of the answer.

Generation Mode

Use --mode instruction when the checkpoint was trained on this format. It wraps the prompt as: 

<BOS><USER>
prompt text
<ASSISTANT>

Use --mode raw only for plain text continuation datasets.

Current Limits

  • The built-in parser expects single-turn examples.
  • Do not put literal control markers such as <BOS>, <USER>, <ASSISTANT>, or <EOS> inside prompt or answer content.
  • Exact-match evaluation works best for short factual, arithmetic, JSON, and formatting tasks.
  • Open-ended assistant answers need looser evaluation modes that are not implemented yet.
  • Multi-turn dialogue will require a deliberate session format and more training data.

Validate Before Training

Use validate-data before training or evaluation to catch malformed examples and inspect dataset size: 

./bin/mgpt validate-data --input data.txt

The command reports byte count, example count, marker counts, prompt length stats, and answer length stats. It returns a nonzero exit code if required markers are missing or unbalanced.

The unit tests also enforce a round-trip contract: canonical text, JSONL import, and deterministic splitting all flow back through this same validation path.

Split Before Training

Use split-data to create deterministic train and validation files from one instruction dataset: 

./bin/mgpt split-data \
  --input data.txt \
  --train train.txt \
  --val val.txt \
  --ratio 0.9 \
  --seed 42

Then train with --val-input val.txt so validation always measures held-out examples instead of an implicit token slice.

JSONL Import Format

JSONL is easier to generate, validate with external tools, shuffle, and split. Each line should contain a user string and an assistant string: 

{"user":"What is 2+2?","assistant":"4"}
{"user":"Return JSON with key answer and value 42.","assistant":"{\"answer\":42}"}

Convert JSONL into the native instruction format with:

./bin/mgpt import-jsonl \
  --input pairs.jsonl \
  --output data.txt

The tracked assistant seed dataset follows this path from sample_data/assistant/seed.jsonl into generated files under artifacts/datasets/assistant/.

Reliability Rule

  • If validate-data fails, the dataset should not be used for training or evaluation.
  • All assistant-style examples should be converted into the canonical single-turn format before use.
  • Any new dataset tooling should preserve the same marker order and line structure.
