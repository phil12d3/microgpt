Current Direction

1. Keep the lab harness

Arithmetic and synthetic tasks give clear pass/fail signals when code changes affect training or generation.

2. Build toward a tiny assistant

Assistant behavior needs curated instruction data, richer evaluation, and eventually more context support.

3. Document every feature as it lands

Code changes should be paired with updates under docs/ so usage, tradeoffs, and data expectations stay visible.

Quick Start

make test
make cli-test
make strict-test
make sanitize-build
./bin/mgpt make-arithmetic-data --output /private/tmp/microgpt-train.txt --max-a 9 --max-b 9
./bin/mgpt train --input /private/tmp/microgpt-train.txt --checkpoint /private/tmp/microgpt.bin --steps 1000
./bin/mgpt generate --checkpoint /private/tmp/microgpt.bin --prompt "What is 2+2?" --mode instruction --greedy --quiet

strict-test builds with warnings as errors. sanitize-build compiles with address and undefined-behavior sanitizers. cli-test exercises the user-facing data, training, generation, eval, and negative-path commands.
