Public API

The supported embedding boundaries are include/microgpt/api.hpp for C++ and include/microgpt/api_c.h for C. Application code should prefer these facades over CLI-specific headers.

Current Surface

  • microgpt::api::TrainingOptions configures a small training run and can disable the validation split.
  • microgpt::api::GenerationOptions configures generation.
  • microgpt::api::ModelHandle owns a loaded model, optimizer state, and checkpoint step.
  • microgpt::api::load_model loads a checkpoint with a selected backend.
  • microgpt::api::generate runs text generation from a loaded handle.
  • microgpt::api::train_text_file trains from a byte/text file and writes a checkpoint.

Example

#include "microgpt/api.hpp"

int main() {
  auto model = microgpt::api::load_model("model.bin");
  microgpt::api::GenerationOptions options;
  options.max_new_tokens = 16;
  options.top_k = 1;
  std::string text = microgpt::api::generate(model, "Hello", options);
}

The build includes examples/embed_api.cpp as bin/embed_api_example so this public header is compiled by the normal build.

C API

The C API exposes an opaque microgpt_model, checkpoint loading, generation, owned string cleanup, and microgpt_last_error for exception-free error reporting. 

#include "microgpt/api_c.h"

microgpt_model* model = microgpt_load_model("model.bin", "cpu");
microgpt_generation_options options = microgpt_default_generation_options();
char* text = microgpt_generate_text(model, "Hello", options);
microgpt_free_string(text);
microgpt_free_model(model);

The normal build creates bin/libmicrogpt.a and compiles examples/embed_c.c as bin/embed_c_example.

Not Stable Yet

This is the first public API boundary, not a long-term ABI promise. The C API is the preferred direction for future stable embedding, while the C++ API remains the preferred source-level integration point.
