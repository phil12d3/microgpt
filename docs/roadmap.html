Roadmap

The project direction is a tiny assistant, supported by lab-style tasks that keep training and generation behavior measurable.

Guiding Position

Toy Lab

Controlled tasks such as arithmetic and formatting are used to detect regressions, compare scaling runs, and confirm code changes behave as expected.

Tiny Assistant

The useful end goal is assistant-like behavior: instruction following, concise answers, formatting, and eventually context.

Completed Foundations

  • Portable C++17 model, training, checkpointing, and generation.
  • Instruction-format tokenizer with special control tokens.
  • Generation prompt alignment fix and regression test.
  • Arithmetic data generation and exact-match evaluation.
  • Explicit validation file support.
  • Instruction dataset validation, splitting, and JSONL import.
  • Evaluation modes for exact, prefix, and contains matching.
  • Checkpoint metadata sidecars with dataset paths, command details, and config.
  • Checkpoint metadata history with model size, timing, and per-run data paths.
  • Reusable harness registry and CLI dispatcher.
  • Separate executables for mgpt, lab, and chat.
  • Core library split into concern-based headers with microgpt.hpp kept as an aggregate include.
  • Reusable chat harness in include/microgpt/chat.hpp.
  • Generated artifacts organized under artifacts/.
  • Static HTML documentation under docs/.

Recommended Next Phase

  1. Add a backend abstraction so CPU remains the reference while Metal and CUDA can be added cleanly.
  2. Benchmark the current CPU path and use it as the baseline for accelerated backends.
  3. Improve assistant single-turn memorization and validation behavior with controlled experiments.
  4. Consider multi-turn session context after single-turn behavior is reliable.

Tradeoffs

  • More data improves assistant behavior only when the examples encode the desired behavior clearly.
  • Longer context enables richer prompts but slows training and increases data requirements.
  • A harness layer reduces CLI coupling, but it adds another integration surface to maintain.
  • Metal and CUDA can improve throughput, but they require strict CPU parity tests to avoid hiding correctness regressions.
  • Multi-turn context is useful, but it complicates prompt formatting and evaluation.
  • Exact-match tests are narrow but reliable; assistant-quality tests are broader but less crisp.
  • Keeping generated files organized makes experiments easier to compare, but it introduces retention and naming policy work.
