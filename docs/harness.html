Harness

The harness layer lets other codebases embed microgpt tools without going through the CLI. The CLI is now just one consumer of the registry.

Headers

  • include/microgpt/harness.hpp for the tool registry and I/O wrapper.
  • include/microgpt/tools.hpp as the composition header for built-in commands.
  • include/microgpt/cli_*.hpp for command-specific reusable implementations.
  • include/microgpt.hpp as the aggregate compatibility header for the core library.
  • include/microgpt/core.hpp, model.hpp, training.hpp, generation.hpp, and related headers for concern-based embedding.
  • include/microgpt/chat.hpp for the reusable interactive chat harness and single-turn prompt formatting.

Embedding Example

#include "microgpt/harness.hpp"
#include "microgpt/tools.hpp"

int main() {
  microgpt::HarnessRegistry registry;
  microgpt::register_microgpt_tools(registry);
  microgpt::HarnessIO io{std::cout, std::cerr};
  return registry.run("validate-data", {"--input", "data.txt"}, io);
}

A matching standalone example lives in examples/embed_harness.cpp.

Embedding Choices

  • Use include/microgpt/tools.hpp when another program wants CLI-style tools such as training, evaluation, data validation, or artifact listing.
  • Use include/microgpt/generation.hpp with checkpoint.hpp when an application wants direct model loading and text generation.
  • Use include/microgpt/chat.hpp when an application wants the same single-turn chat prompt formatting as bin/chat.
  • Use the smaller concern headers instead of include/microgpt.hpp when embedding code only needs one layer of the system.

Custom Tools

Applications can register their own tools next to the built-ins. Each tool receives arguments plus output and error streams, then returns a process-style status code. 

microgpt::HarnessRegistry registry;
microgpt::register_microgpt_tools(registry);

registry.register_tool("hello", [](const std::vector<std::string>&, microgpt::HarnessIO& io) {
  io.out << "hello from host app\n";
  return 0;
});

Current State

  • The CLI uses the same registry as embedders.
  • Command families are split into reusable headers under include/microgpt/.
  • A standalone embedding example is included under examples/.
