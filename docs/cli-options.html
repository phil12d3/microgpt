Reference

microgpt CLI Options

A local reference for the project executables. Use ./bin/mgpt for model operations, ./bin/lab for lab checks, and ./bin/chat for interactive assistant-style use. In a terminal it opens the scrollback UI; otherwise it falls back to simple line input.

Commands

train

Starts a new model from random weights and trains it from an input text file. Existing checkpoint files with the same name are replaced. 

./bin/mgpt train --input data.txt --checkpoint model.bin [options]

resume

Loads an existing checkpoint, keeps its architecture and optimizer state, then continues training for more steps. 

./bin/mgpt resume --input data.txt --checkpoint model.bin [options]

generate

Loads a checkpoint and samples new text from a prompt. Token progress is printed to stderr, and generated text is printed to stdout. 

./bin/mgpt generate --checkpoint model.bin --prompt "text" [options]

lab

Runs built-in sanity checks for gradients, causal masking, and simple training behavior. The lab also covers backend parity and IO/tokenizer safety contracts. 

./bin/lab

cli-test

Runs command-level smoke tests for data generation, validation, splitting, JSONL import, training, generation, eval reports, artifact listing, and expected failure paths. 

make cli-test

eval

Generates answers for instruction-format examples and reports exact-match accuracy. Pass --output to write a JSON report under artifacts/evals/. 

./bin/mgpt eval --checkpoint model.bin --input eval.txt [options]

make-arithmetic-data

Writes reproducible arithmetic instruction examples for controlled training and evaluation experiments. 

./bin/mgpt make-arithmetic-data --output data.txt [options]

validate-data

Checks instruction-format examples, or session-format examples when --format session is selected, reports counts and length stats, and returns a nonzero status for malformed data. 

./bin/mgpt validate-data --input data.txt [--format session]

split-data

Shuffles instruction examples with a seed, or session examples when --format session is selected, and writes explicit train and validation files. 

./bin/mgpt split-data --input data.txt --train train.txt --val val.txt [--format session] [options]

import-jsonl

Converts JSONL records with user and assistant fields into instruction-format examples, or session JSONL with a turns array when --format session is selected. 

./bin/mgpt import-jsonl --input pairs.jsonl --output data.txt [--format session]

list-artifacts

Lists generated artifact files and sizes.

./bin/mgpt list-artifacts --root artifacts

clean-artifacts

Dry-runs artifact cleanup by default. Add --yes to delete files.

./bin/mgpt clean-artifacts --root artifacts --yes

backends

Reports compiled and runtime backend availability.

./bin/mgpt backends

parity

Runs a staged CPU-vs-Metal comparison trace and prints the first divergent stage when the trace drifts.

./bin/mgpt parity --backend metal

bench

Runs a small synthetic training/generation benchmark or an operation-level resident linear benchmark.

./bin/mgpt bench --backend cpu
./bin/mgpt bench --backend metal --resident-linear --iterations 1000 --rows 128 --in 128 --out 128

eval

Evaluates instruction-format examples by default, or session examples when --format session is selected. 

./bin/mgpt eval --checkpoint model.bin --input eval.txt [--format session] [options]

Training Options

Option Commands Default Purpose
--input PATH train, resume Required Text file used as the training corpus.
--val-input PATH train, resume Automatic split Explicit validation file. When provided, all of --input is used for training.
--split-ratio VALUE train, resume 0.9 Controls the automatic validation split when --val-input is not provided.
--no-val-split train, resume Off Trains and validates on the full input file instead of creating a split.
--checkpoint PATH All except test checkpoint.bin for training, required for generation Binary model checkpoint to write, resume, or load.
--steps N train, resume 1000 Number of optimizer steps to run in this invocation.
--context N train 64 Sequence length in tokens. Smaller corpora need smaller context sizes.
--tokenizer byte|bpe train byte Tokenizer used to create a new checkpoint. byte is the legacy byte-level mode. bpe uses the built-in deterministic multi-character vocabulary. Resume and generation read the tokenizer from the checkpoint.
--d-model N train 128 Transformer hidden width. Larger values increase capacity and runtime.
--layers N train 2 Number of transformer blocks.
--heads N train 4 Attention heads per block. --d-model must be divisible by this value.
--ff N train 512 Feed-forward hidden width inside each transformer block.
--batch-size N train 16 Number of context windows sampled per training step.
--lr VALUE train 0.0003 AdamW learning rate.
--eval-interval N train 500 How often to compute validation loss and perplexity.
--save-interval N train 1000 How often to write the checkpoint during training.
--progress-interval N train, resume 10 How often to print progress lines.
--seed N train 42 Random seed for initialization and batch sampling.

Resume behavior

resume loads model shape and optimizer state from the checkpoint. Architecture flags such as --context, --d-model, --layers, --heads, --ff, and --tokenizer are ignored after the checkpoint is loaded. Passing a different explicit tokenizer while resuming is rejected.

Generation Options

Option Commands Default Purpose
--checkpoint PATH generate Required Checkpoint to load before generating text.
--prompt TEXT generate Required unless --prompt-token is used Text prompt to condition generation.
--prompt-token N generate None Starts generation from a single byte token in the range 0..255.
--mode instruction generate instruction Wraps the prompt as <BOS><USER>\n... \n<ASSISTANT>\n.
--mode raw generate None Sends exactly the prompt text to the model without instruction tags.
--max-new-tokens N generate 200 Maximum number of tokens to sample after the prompt.
--temperature VALUE generate 0.8 Sampling randomness. Lower values are more deterministic; higher values are more varied.
--top-k N generate 20 Samples only from the N highest-probability next tokens. Use 1 for greedy output.
--greedy generate, eval Off Shortcut for greedy decoding. Equivalent to --top-k 1.
--quiet generate Off Suppresses token progress logs so stdout contains only generated text.

Evaluation Options

Option Commands Default Purpose
--checkpoint PATH eval Required Checkpoint to evaluate.
--input PATH eval Required Instruction-format examples containing expected answers.
--max-examples N eval All examples Limits evaluation to the first N parsed examples.
--hide-failures eval Off Prints only the final accuracy summary.
--output PATH eval None Writes a JSON report with settings, accuracy, and per-example results.
--match MODE eval exact Scoring mode: exact, prefix, or contains.
--max-new-tokens, --temperature, --top-k, --greedy eval 200, 0.2, 1 Controls generation during exact-match evaluation.

Validation Options

Option Commands Default Purpose
--input PATH validate-data Required Instruction-format dataset to validate and count.

Splitting Options

Option Commands Default Purpose
--input PATH split-data Required Instruction-format dataset to split.
--train PATH split-data Required Output path for shuffled training examples.
--val PATH split-data Required Output path for held-out validation examples.
--ratio VALUE split-data 0.9 Fraction of examples written to the training file.
--seed N split-data 42 Deterministic shuffle seed.

Option Commands Default Purpose
--input PATH import-jsonl Required JSONL file containing one object per line.

Backend Options

Option Commands Default Purpose
--backend MODE train, resume, generate, eval, chat cpu Selects cpu, metal, or cuda. Metal requires a binary built with make BACKEND=metal; incomplete operations use CPU fallback. 
make deps
make BACKEND=metal all
./bin/mgpt backends
./bin/mgpt parity --backend metal
./bin/mgpt train --backend metal --input data.txt --checkpoint model.bin

Benchmark Options

Option Commands Default Purpose
--resident-linear bench off Runs repeated linear forward/backward work with persistent backend buffers. Metal uploads once, batches the repeated work into one command buffer, and downloads once when runtime Metal is available.
--iterations N bench --resident-linear 100 Number of resident linear forward/backward iterations to run.
--rows N, --in N, --out N bench --resident-linear 128 Controls the synthetic linear operation shape.

Benchmark output includes backend upload/download counts and bytes. Resident Metal should show a small fixed number of transfers; normal model benchmarks currently expose the remaining synchronization cost. 

make BACKEND=metal all
./bin/mgpt bench --backend metal --resident-linear --iterations 1000 --rows 128 --in 128 --out 128
./bin/mgpt bench --backend cpu --resident-linear --iterations 1000 --rows 128 --in 128 --out 128

Checkpoint Metadata

Training writes a binary checkpoint and a JSON sidecar at the same path with a .json extension. For example, model.bin writes model.json. The binary file is used for loading model weights. The JSON file is for inspection and experiment tracking.

Field Purpose
step, optimizer_step Final checkpoint step and AdamW optimizer step.
start_step, requested_steps Resume start point and number of steps requested for that run.
command The CLI command used for the final training run.
parameter_count, parameter_bytes_f32 Model size as parameter count and approximate float32 weight bytes.
train_input, val_input Dataset paths used for training and validation.
validation_source explicit when --val-input was used, otherwise auto_split or full_data when --no-val-split was used.
config Model and optimizer hyperparameters stored for the checkpoint.
training_history One entry per train or resume run, including timing, data paths, steps, command, model size, and final observed losses.

Option Commands Default Purpose

Examples

Train a small model for the current tiny dataset

./bin/mgpt train \
  --input data.txt \
  --checkpoint model.bin \
  --steps 1000 \
  --context 32 \
  --d-model 32 \
  --layers 1 \
  --heads 4 \
  --ff 64 \
  --batch-size 2 \
  --lr 0.001 \
  --eval-interval 50 \
  --save-interval 100 \
  --progress-interval 10

Continue training an existing checkpoint

./bin/mgpt resume \
  --input data.txt \
  --checkpoint model.bin \
  --steps 500 \
  --progress-interval 10

Generate from instruction-formatted data

./bin/mgpt generate \
  --checkpoint model.bin \
  --prompt "What is 2+2?" \
  --mode instruction \
  --max-new-tokens 80 \
  --temperature 0.2 \
  --top-k 1

Hide token progress logs

./bin/mgpt generate \
  --checkpoint model.bin \
  --prompt "What is 2+2?" \
  --mode instruction \
  --max-new-tokens 80 \
  --temperature 0.2 \
  --top-k 1 \
  --quiet

Train with an explicit validation file

./bin/mgpt train \
  --input train.txt \
  --val-input val.txt \
  --checkpoint model.bin \
  --steps 3000 \
  --context 64 \
  --d-model 64 \
  --layers 2 \
  --heads 4 \
  --ff 128 \
  --batch-size 8

Generate arithmetic experiment data

./bin/mgpt make-arithmetic-data \
  --output arithmetic.txt \
  --max-a 19 \
  --max-b 9

Validate instruction data

./bin/mgpt validate-data \
  --input arithmetic.txt \
  [--format session]

Split instruction data

./bin/mgpt split-data \
  --input arithmetic.txt \
  --train arithmetic-train.txt \
  --val arithmetic-val.txt \
  [--format session] \
  --ratio 0.9 \
  --seed 42

Import JSONL assistant examples

./bin/mgpt import-jsonl \
  --input pairs.jsonl \
  --output assistant-data.txt \
  [--format session]

Evaluate exact-match accuracy

./bin/mgpt eval \
  --checkpoint model.bin \
  --input arithmetic.txt \
  --max-new-tokens 20 \
  --match exact \
  --greedy \
  --hide-failures \
  [--format session] \
  --output artifacts/evals/arithmetic-eval.json

Practical Tips

Match the prompt mode to the data

Use --mode instruction for files that look like <BOS><USER>...<ASSISTANT>.... Use --mode raw for plain continuation text.

Small data limits output quality

A few hundred bytes can demonstrate memorization, but it cannot teach broad question answering. More examples are needed for useful instruction behavior.

Watch both losses

Falling train loss means the model is fitting the corpus. Rising validation loss means it is memorizing rather than generalizing.

Use greedy output for checks

When testing whether a tiny model learned a specific example, use --temperature 0.2 --top-k 1 to reduce randomness.
