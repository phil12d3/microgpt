Artifacts

Generated datasets and checkpoints live under artifacts/ so experimental files do not accumulate in the repository root.

Layout

  • artifacts/datasets/ for generated or temporary training and validation files.
  • artifacts/checkpoints/ for generated model checkpoints and metadata sidecars.
  • artifacts/evals/ for generated evaluation reports.
  • sample_data/ for small curated examples that are intended to stay in git.

Checkpoint Files

The .bin file is the actual model checkpoint and is required for loading, resuming, generating, evaluating, and chat. The .json file is metadata only.

Metadata sidecars record config, model size, dataset paths, command details, and a training_history entry for each training or resume run that rewrites the same checkpoint path.

Evaluation Reports

Evaluation reports are optional JSON files written by eval --output. They record the checkpoint, dataset, scoring mode, generation settings, accuracy, and each evaluated case. 

./bin/mgpt eval \
  --checkpoint artifacts/checkpoints/model.bin \
  --input artifacts/datasets/eval.txt \
  --match exact \
  --greedy \
  --output artifacts/evals/model-eval.json

Example Paths

./bin/mgpt train \
  --input sample_data/data.txt \
  --checkpoint artifacts/checkpoints/model.bin \
  --steps 1000

./bin/chat --checkpoint artifacts/checkpoints/model.bin

Listing and Cleanup

Use list-artifacts to inspect generated files and clean-artifacts to remove them. Cleanup is a dry run unless --yes is passed. 

./bin/mgpt list-artifacts --root artifacts

./bin/mgpt clean-artifacts --root artifacts

./bin/mgpt clean-artifacts --root artifacts --yes
